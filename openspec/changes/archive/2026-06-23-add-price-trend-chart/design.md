## Context

目前系統已具備價格資料新增、查詢、刪除、optional crawler 匯入與 SQLite 持久化。首頁以表格呈現資料，但缺少視覺化趨勢，也缺少修改既有紀錄的能力。這個 change 會在不改變資料表結構的前提下，新增首發價格趨勢折線圖，並補上價格紀錄編輯流程。

## Goals / Non-Goals

**Goals:**

- 在首頁顯示首發價格趨勢折線圖。
- 使用既有 `GET /api/prices` 作為圖表資料來源。
- 只將包含 `release_date` 與有效 `price` 的紀錄顯示在圖表中。
- 新增 `PUT /api/prices/:id` 修改既有價格紀錄。
- 編輯資料時沿用新增資料的必填欄位、日期格式與價格格式驗證。
- 修改成功後重新載入表格與折線圖。

**Non-Goals:**

- 不新增資料庫欄位或改變 `game_prices` schema。
- 不新增登入、會員、圖表篩選器、部署或測試框架。
- 不改變 optional crawler 的匯入邏輯。
- 不新增大量資料分析或官方 CPI 計算。

## Decisions

- 使用 Chart.js 繪製折線圖，因為前端仍維持原生 HTML、CSS、JavaScript，不引入 React、Vue 或其他前端框架。
- 圖表直接呼叫既有 `GET /api/prices`，避免新增只服務圖表的 API，並保持後端介面簡單。
- 圖表資料在前端過濾與排序，只顯示有 `release_date` 與可用 `price` 的資料，避免缺少上市日期的手動紀錄造成圖表座標不完整。
- Y 軸最低點依目前圖表資料中的最低價格往下取到最接近的 50 倍數，讓首發價格差異更容易觀察，同時避免固定從 0 開始壓縮變化。
- 編輯功能使用 `PUT /api/prices/:id`，符合以 id 更新完整資源的語意，並與既有 `DELETE /api/prices/:id` 路由一致。
- 後端更新資料時不接受前端覆蓋 `data_type`，manual/crawler 的來源語意應由後端與 crawler 流程維護。
- 前端在表格中提供編輯操作，使用既有欄位進行修改，降低 UI 複雜度並維持期末專題 MVP 範圍。

## Risks / Trade-offs

- Chart.js 透過 CDN 載入時，離線環境可能無法顯示圖表。→ 若展示環境不能連網，可改為下載本地檔案，但本 change 先維持簡單導入。
- 前端過濾圖表資料代表表格筆數可能多於圖表點位。→ 使用訊息顯示目前繪製的有效資料筆數，讓使用者知道圖表只包含有上市日期的資料。
- 前端若直接使用既有表單編輯，使用者可能不清楚目前是新增或編輯模式。→ 前端應在進入編輯模式時改變按鈕文字或顯示狀態訊息。
- 更新 crawler 匯入資料可能改變其原始來源紀錄。→ 編輯時保留既有 `data_type`，不讓前端覆蓋來源類型。
